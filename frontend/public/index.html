Drop‑in Components

Tonika lets you drop *onika modules into cards with one attribute: 

<div class="tk-card" data-onika="Clavonika" data-target="#clavonika-slot"></div>

No frameworks, no build steps — just include the module’s script and Tonika mounts it.

Clavonika (Piano)

Requires window.Clavonika

Jackonika (MIDI‑In)

Requires window.Jackonika

Components

  • Clavonika — Piano keyboard UI
  • Jackonika — MIDI‑in stompbox / monitor
  • Backplanika — Micro router & patching
  • Midonika — MIDI visualizer

Each module exposes a global with a mount(el, opts) method.

Rack Sizing (U & HP)

Tonika treats your browser window like a modular rack. Cards are sized using the same ideas as Eurorack: vertical units (U) and horizontal pitch (HP).

Unit Meaning in Tonika
1U ~48px tall row
3U ~1 module section
6U ~2 module sections
HP ~12px wide column
42HP Small rack width (mobile)
60HP Medium rack width
84HP Large rack width (desktop)

How to size a card

  1. Add data-u for height (in U).
  2. Add data-hp for width (in HP).
  3. Tonika JS auto-converts them into CSS classes (u-X hp-Y). 
      <article class="tk-card" data-u="3" data-hp="20">
        ...
      </article>

This makes it easy to grow your rack as new *onika modules appear.
