Drop-in Components

Tonika lets you drop *onika modules into cards with one attribute: 

<div data-onika="Clavonika" data-target="#clavonika-slot"></div>

No frameworks, no build steps — just include the module’s script and Tonika mounts it.

Clavonika (Piano)

Requires window.Clavonika

Jackonika (MIDI-In)

Requires window.Jackonika

Backplanika (Router)

Requires window.Backplanika

Midonika (Visualizer)

Requires window.Midonika

Components

Components

  • Clavonika — Piano keyboard UI
  • Jackonika — MIDI-in stompbox / monitor
  • Backplanika — Micro-router & patching
  • Midonika — MIDI visualizer

Each module exposes a global with a mount(el, opts) method.

Rack Sizing

Tonika treats your browser window like a modular rack. Cards are sized using Small, Medium, and Large, static (traditional html) and Dynamic adjusting to inserted component.

Card Meaning in Tonika
Small 20
Medium 40
Large 80

Rack Width

Maximum 80 wide, so 1 large component, 2 medium 40 components (80), 4 small (80), or mixtures of to maximum of 80

The rack always fits within the top menu width — no horizontal page scrolling.
