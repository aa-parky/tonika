Demonstrating the new configurable architecture and flexible sample mappings

🔧 Refactoring Overview

Key Changes:
  • Hard-coded Sample Mappings: removed from source; now configured via constructor options.
  • Global namespace: legacy shim removed; proper module/one global export.
New Benefits:
  • CDN-friendly while working with local Tonika directory structures
  • Runtime kit switching and custom sample mappings
  • Modern ES6 module support with backward compatibility
  • Flexible configuration system for different deployment scenarios
